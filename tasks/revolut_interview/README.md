# Revolut Android Live Coding Interview Practice

## Rules (as on real interview)

1. **Time limit: 60 minutes per task** — set a timer
2. **No Copilot, no Google, no StackOverflow** — close everything
3. **Talk out loud** — explain every decision as if interviewer is listening
4. **Write tests** — they expect unit tests as part of the solution
5. **Views/XML only** — no Jetpack Compose
6. **SOLID principles** — clean code, not hacky solutions
7. **Each task has follow-up questions** — prepare verbal answers

## Coding Tasks

Create a new empty Android project in Android Studio, read only Part 1, and go.

| # | Task | Key Skills |
|---|------|-----------|
| 1 | Transaction History | RecyclerView, DiffUtil, filtering, search, debounce |
| 2 | Money Transfer | Form validation, state management, error handling |
| 3 | Contacts with Search | Pagination, debounce, flatMapLatest, selection mode |

## Theory (oral questions during/after coding)

| File | Topics |
|------|--------|
| THEORY_CONCURRENCY.md | Coroutines, Dispatchers, JMM, deadlocks, memory leaks, Android threading |
| THEORY_TESTING.md | ViewModel testing, MockK, Turbine, runTest, debounce testing |

## How to practice

1. Read ONLY Part 1 of a task
2. Implement + write tests
3. Then read Part 2 — simulates interviewer changing requirements
4. Then Part 3
5. Check time — did you fit in 60 minutes?
6. Review follow-up questions — prepare verbal answers
7. Read theory files and practice answering out loud
